# Lab14 — Grunge Music Utils

Утилиты для анализа "грязных" рок-плейлистов: поиск самого громкого трека и вычисление гранж-индекса.

## Цель работы
Освоить методы отладки, модульного тестирования (Jest) и анализ работы функций.

## Пользователь может:  
- выбирать трек из плейлиста и считать его **гранж-индекс**;  
- находить **самый громкий трек**;  
- использовать **рандомайзер**, чтобы получить случайный трек с индексом;  
- наблюдать **имитацию тестов**, где показана ошибка расчёта;  
- **исправлять ошибку** и видеть, как тесты проходят успешно;  
- добавлять свои треки с параметрами громкости, искажений, длительности и шума.  

### getLoudestTrack(tracks)
Возвращает трек с максимальным уровнем громкости.

### calculateGrungeScore(track)
Вычисляет условный "гранж-индекс":
- длительность трека
- уровень дисторшна
- уровень шума

(формулу можно отлаживать в DevTools)

## Установка зависимостей
npm install

## Запуск тестов
npm test

## Использование
- **Выбрать трек** → нажать «Посчитать гранж-индекс»  
- **Рандомайзер** → нажать «Give me something nasty»  
- **Исправление ошибки** → нажать «Исправить ошибку»  
- **Тесты** → отображаются в блоке «Тесты» при расчёте гранж-индекса  

### Примеры входных данных
| Название трека         | Громкость | Длительность | Искажения | Шум |
|------------------------|-----------|--------------|-----------|-----|
| Negative Creep         | 96        | 184          | 9         | 10  |
| Blew                   | 80        | 162          | 6         | 4   |
| Rain When I Die        | 91        | 350          | 5         | 7   |

### Примеры выходных данных
- **Гранж-индекс трека "Negative Creep" до исправления ошибки:** 211.0 (не учитывались искажения)  
- **Гранж-индекс трека "Negative Creep" после исправления ошибки:** 229.9
